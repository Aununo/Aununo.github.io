对称加密

对称加密是使用相同密钥来加密和解密数据的算法。目标是使用短密钥安全有效地发送长消息。

最著名的对称密钥密码是高级加密标准 (AES)，于 2001 年标准化。它的应用如此广泛，以至于现代处理器甚至包含特殊的指令集来执行 AES 操作。这里的第一系列挑战将引导您了解 AES 的内部工作原理，向您展示其单独的组件如何协同工作以使其成为安全的密码。最后，您将构建自己的代码来执行 AES 解密！

我们可以将对称密钥密码分为两种类型：分组密码和流密码。分组密码将明文分解为固定长度的块，并通过加密函数与密钥一起发送每个块。同时，流密码通过将伪随机密钥流与数据进行异或，一次加密一个字节的明文。 AES 是一种分组密码，但可以使用 CTR 等操作模式转换为流密码。

分组密码仅指定如何加密和解密单个块，并且必须使用一种操作模式将密码应用于更长的消息。这就是现实世界的实现经常会失败的地方，因为开发人员不理解使用特定模式的微妙含义。剩下的挑战是让你攻击各种模式的常见误用。

本章包含17个挑战，开头四个挑战介绍了流密码乃至整个密码学的主力——异或运算，中间七个挑战讲解了AES的工作原理，最后七个挑战我们与服务器交互进行实践。

XOR starter

XOR 是按位运算符，如果位相同则返回 0，否则返回 1。在教科书中，XOR 运算符用 ⊕ 表示，但在大多数挑战和编程语言中，您会看到使用符号 ^ 。

XOR Properties

在上一个挑战中，您了解了异或如何在位级别上工作。在本教程中，我们将介绍 XOR 运算的属性，然后使用它们来撤消已加密flag的一系列操作。当您稍后攻击真正的密码系统时，尤其是在分组密码类别中，获得对其工作原理的直觉将会有很大帮助。

当我们使用 XOR 运算符解决挑战时，我们应该考虑四个主要属性：

Commutative: A ⊕ B = B ⊕ A
Associative: A ⊕ (B ⊕ C) = (A ⊕ B) ⊕ C
Identity: A ⊕ 0 = A
Self-Inverse: A ⊕ A = 0

让我们来分解一下。可交换意味着 XOR 运算的顺序并不重要。结合意味着一系列操作可以无顺序地进行（我们不需要担心括号）。identity是 0，因此与 0 进行异或“什么也不做”，最后与自身进行异或的结果返回零。

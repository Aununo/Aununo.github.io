Introduction

  • 课程目标：
  1. 讲授密码学基本原型(crypto primitives)是如何工作的；
  2. 如何正确使用密码学原型；
  3. 推导构建加密机制的安全性，破解不安全的加密机制。
  • 建议：
  1. 听讲座的时候记笔记，用自己话归纳讲演的内容；
  2. 时常暂停，理清讲座内容。

密码学无处不在

  • 密码学在计算机领域无处不在，是保护数据的常见工具。
  1. 安全通信：网页流量用https协议保护，无线流量wifi用802.11iWPA2保护，手机流量用GSM加密保护，蓝牙流量也用密码学保护，等等。
  2. 文件加密：可以加密存储在硬盘上的文件，硬盘被盗文件也不会被窃取。EFS，TrueCrypt.
  3. 内容保护：DVD和蓝光碟上面的电影是被加密的。DVD使用的是CSS系统，即内容混淆系统；蓝光碟使用AACS系统。CSS是一个容易破解的系统。
  4. 认证加密
  • 安全通信——TLS协议

Alice在笔记本端，Bob在服务器端，他们想实现安全通信，使用的是HTTPS协议，实际上使用SSL，有时用TLS，这三个是不同的协议。确保数据在网络传输中，攻击者无法窃听数据内容，并且无法篡改。TLS主要由两部分组成：

  1. 握手协议：双方互相交谈，在握手的最后，双方分享一个密钥，但攻击者不知道密钥。这是第二部分所讲的公钥加密。
  2. Record Layer：如何用共享的密钥加密传输以保护通信数据？这是第一部分所讲的内容。
  • 文件加密

一个已加密的文件，攻击者无法读取文件数据。如果其试图改动文件数据，当Alice解密文件时，就会检测出改动。所以我们有文件的私密性和完整性。从哲学角度，这里的文件加密很像今天的Alice和明天的Alice之间的安全通信。

注：

  1. 加密和解密的算法实际上是公开的，所有人都知道，仅有共享的密钥是保密的。你应当只使用公开的加密算法。很多私有的加密算法一旦被逆向工程，就很容易被分析破解。
  2. 加密电子邮件使用一次密钥，但加密文件系统里的文件同一密钥用来加密很多文件。
  3. 密码学不能解决所有安全问题。WEP是一个有很多错误的协议。你只需使用密码学标准，而不能随意发明。

什么是密码学

密码学的核心是安全通信，首先是安全密钥的建立，然后是共享密钥，同时Alice需要知道她在和Bob对话，而Bob知道和他对话的人是Alice.

  • 数字签名

数字签名是待签内容的一个函数值，因为新文件的函数值和旧签名不一致，所以攻击者无法复制粘贴签名。

  • 匿名通信

有一个标准mix net，允许Alice和公共服务器Bob对话，经过一系列代理，Bob无从得知对方的身份。有趣的是，匿名通信是双向的，尽管Bob不知道他在和谁说话，但他仍然可以回应Alice，Alice可以收到回复。

  • 匿名数字现金

如果Alice可以花掉虚拟的一美元且商家不知道她是谁，那么Alice就可以复制这一美元，无限消费。怎么样才能既匿名又避免复制货币呢？确保如果Alice只花费一次，没人知道她是谁；如果她花费多次，她的个人信息将会被全部暴露。

  • 多方安全计算

参与者自己有一个秘密输入，计算出函数值。怎样既能得到函数值，又对各个输入进行保密。当然可以引入可信第三方。

  1. 投票问题：在不知道每个人投票结果的情况下，计算多数的投票。引入一个可信第三方选举中心，选民们将投票加密后投给选举中心，由中心计算出胜者。
  2. 私密拍卖：采用Vickrey拍卖机制，即出价最高的人中标，但是付的钱是第二高的人出的价。问题是如何计算谁出钱以及出多少钱，但其余的信息完全保密？最高出价保密，公众只知道第二高的数额和最高出价者。

密码学有一个十分核心的定理，对于计算这样的函数，只要你能用可信第三方计算，就一定能不借助可信任方完成。

各参与者使用某种协议互相通信，在协议的最后这个函数值突然为大家所知，除此之外无任何信息泄露。

  • 私有外包计算

假设Alice有一个搜索请求query，事实上有很特殊的加密机制E(query)可以把Alice的请求加密后交给Google，然后Google可以在加密后的值上运算而无需知道原文是什么。Google运行庞大的搜索算法，得到加密后的结果，并再发回给Alice，解密后就可以得到她想要的结果。但目前这只是理论结果。

  • 零知识证明

Alice知道一个大整数N，由两个大的素数相乘。她知道N的因式分解，零知识证明就是Alice在不告诉Bob这两个素数的前提下，向Bob证明自己知道N的因式。再比如你有一个数独谜题，你可以像Bob证明你知道解但不告诉Bob解是什么。

  • 现代密码学

现代密码学是很严格的科学。我们描述的每一个概念都反复用到严格的三步：

  1. 准确描述威胁模型是什么；
  2. 提出架构；
  3. 证明任何攻击者在这个威胁模型下可以攻击这个架构，这个攻击者也可以用来解决根本性难题，如果这个问题真的很难，实际上就证明了攻击者无法在威胁模型下破解这个架构。

密码学的历史

《The code breakers》David Kahn，1996

  • 对称加密

Alice和Bob共享一个相同的密钥k，Alice以明文m作为输入，密码是一对算法，包括加密E和解密D. Alice将明文加密成密文c := E(k, m)，Bob将c进行解密得到m := D(k, m).

我们还会看到加解密密钥不同的情况，这里我们学习历史上都已被破解的对称加密。

  • 替换式密码

最简单的替换式密码的密钥是一张替换表，将字母一一映射到其它字母。

  • Caesar密码 (罗马时期)

通过偏移量，这是一个固定密钥，或者可以说没有密钥。

  • 如何破解替换式密码？

密钥空间有多大？对于每一个英文字母，在此字母集合上的映射一共有26×25×···×2×1即26!个，即对密钥空间内的元素进行全排列，约为2的88次方，也就意味着描述一个替换密钥需要88位，88比特。这个大小对于密钥空间是合适的。但它仍然是不安全的。

  1. 使用字母的频率来破解：在英文文献中，E是最常见的字母，出现频率约为12.7%；其次是字母T，出现频率约为9.1%；然后是A，频率约为8.1%。
  2. 使用字母对的频率来破解：he，an，in，th

替换式密码对最艰苦的攻击形式：惟密文攻击是脆弱的。仅凭密文，攻击者就能还原整个密钥。

  • Vigener密码 (文艺复兴时期)

密钥是一个单词，比如k = crypto，把明文写在密钥下面，然后不断重复密钥以覆盖整个信息。最后将密钥字母和信息字母相加，对26取模，得到密文。

  • 破解Vigener密码

惟密文攻击。假设密钥的长度已知为6，我们将密文每6个字母分为一组，观察每一组的每一位。因为最常见的字母是E，所以假设每组密文中第一位出现频率最高的字母是H，就可推测H是E的加密，即密钥的第一位是C，因为3 + 5 = 8. 以此类推，还原整个密钥。即使不知道密钥的长度，通过不断尝试长度，也可以快速通过上面的过程进行快速破解。

  • 轴轮机 (电气时代)
  1. 单轮Hebern机：每按下一次字母，密钥表会进行旋转。同理进行破解。
  2. 多轮Enigma机：多个轮子转动的速率不同，四轮机的密钥空间为26的4次方，约为18位，比较小。现在你可以快速借助计算机暴力穷举。英国密码学家Bletchley Park曾发起过对Enigma机的惟密文攻击，在二战时能破译德军的密码。
  • DES (数字时代1974)

1974年IBM提出的DES成为联邦数据加密标准。DES的密钥空间有56位，现在已经很小，但在当时很大。这个加密标准一次加密8个字母，64位。

如今2001年有高级加密标准AES，2008年有Salsa20。

Discrete Probability

Background on discrete probability

许多自然的密码学机制被发现是不安全的，因此，现代密码学是一门严谨的科学，其中的机制总是有其安全性证明。用来描述证明的语言依赖于离散概率。

  1. 离散概率定义在全局U上，U通常为有限集，表示全体n位二进制串的集合，记作{ 0, 1 }^n，有2^n个元素。

  2. 概率分布：在U上的概率分布定义为一个函数P，此函数为全局中的每一个元素赋值，范围在[ 0, 1 ]之间，这个值称为U中特定元素的权重(weight)或概率。函数P仅有一个约束条件，即对所有的元素x，权重和为1. 两个概率分布的经典例子：

    • 均匀分布：P( x ) = 1 / | U |.
    • 点分布：把所有权重都放在一个点上，满足P ( x_0 ) = 1, P( x ≠ x_0 ) = 0.

    分布向量：将U中每个元素的权写出来，将整个分布表示为一个向量。

  3. 事件：考虑U的子集A，Pr[A] = $\sum$P( x $\in$ A) $\in$ [0, 1]. 注意Pr[U] = 1. 我们把集合A叫作一个事件。

  4. 并集上界：Pr[$A_1 \cup A_2$] ≤ Pr[$A_1$] + Pr[$A_2$]. 这是一种“或”关系。

  5. 随机变量：一个随机变量是一个从U到V的映射。如果我们有特定集合V上的随机变量，这个随机变量实际上可推出集合V上的概率分布。Pr[ X = v ] := Pr[ X^{-1}(v) ].

  6. 均匀随机变量：r $\overset{R}{\leftarrow}$ U. let r be a uniform random variable on {0, 1}^2，define the random variable X = r_1 + r_2，then Pr[X = 2] = Pr[r = 11] = 1/4.

  7. 随机算法：y = A(m ; r)，每次算法运行时，r都重新取样。 y$\overset{R}{\leftarrow}$A(m).

  8. 独立性：Pr[A and B] = Pr[A] × Pr[B].

  9. 异或xor：XOR of two strings in {0,1}^n is their bis-wise addtion mod 2.

Stream Cipher

Definition of Symmetric Ciphers

A cipher defined over $(K, M, C)$ is a pair of “efficient” algorithms $(E, D)$ where
$$
E:K \times M \longrightarrow C
$$

$$
D:K\times C\longrightarrow M
$$

and $\forall m\in M,k\in K:$
$$
s.t.\ D(k,E(k,m))=m\ \ \ (1.1)
$$
The equation (1.1) is usually called consistency requirement of a cipher.

What is efficient? Different people has different ideas. If you’re inclined towards theory, efficient means runs in polynomial time. But if you’re more practically inclined, efficient means runs in a certain time period, like within one minute.

E is often randomized, and D is always deterministic. 这意味着，当加密信息时，E 自身生成随机数，它将使用这些随机数加密给定的明文信息。而 D 获取密文作为输入，它的输出是确定的，即明文。

Information Theoretic Security

This is something about information theory. The first person to study the security of ciphers rigorously is very very famous Claude Shannon. In 1949, he published a famous paper Communication Theory of Secrecy Systems which proposed an idea of Perfect Security.

The basic idea of Shannon is that if you’re just an eavesdropper getting to see a single CT, then it should reveal “no” info about the PT.

Def: a cipher $(E,D)$ over $(K,M,C)$​​ has perfect security if

$\forall m_0,m_1\in M,c \in C$ and $len(m_0)=len(m_1):$
$$
Pr[E(k,m_0)=c]=Pr[E(k,m_1)=c]\ \ \ (1.2)
$$
where $k \overset{R}{\leftarrow}K$​.

Comment:

  1. Given CT can’t tell if msg is $m_0$ or $m_1$​​ for all m.
  2. Even the most powerful adv. learns nothing about the PT from CT.
  3. There’s no CT only attack!! But other attacks maybe possible.

One Time Pad (OTP)

The one time pad was invented by Vernam in 1917. It’s the first example of a “secure” cipher.

Is OTP a cipher?

For OTP, $M=K=C={0,1}^n$ and $|M|=|K|=|C|$​. Actually, the key k is a random bit string as long as our msg.
$$
c:=E(k,m)=k \oplus m\ \ \ (1.3)
$$

$\oplus$ is xor which means addition modulo 2. For example, 1 $\oplus$ 1 = 1 + 1 mod 2 $\equiv$ 0 (mod 2). So what the encryption alg does is calculate the addition then modulo 2 bit by bit.

$$
D(k,c)=k\oplus c\ \ \ (1.3)
$$

Indeed, let’s prove that the OTP is satisfied with the consistency requirement.

Proof:
$$
D(k,E(k,m))=k \oplus E(k,m)=k \oplus k \oplus m = 0 \oplus m = m\ \Box
$$

That’s one of the xor properties: k $\oplus$ k = k + k mod 2 = 2k mod 2 = 0.

From a performance point of view, OTP is super super fast. Unfortunately, it’s difficult to use in practice. The length of its key is too long so that if you’re able to transmit your key in a secure way, there’s no necessity to use a cipher, right? Just send your plaintext because it’s the same length with your key k.

Whether it is secure?

Let’s focus a little bit on security.

Lemma: OTP has perfect secrecy.

Proof:

$\forall m \in M,c \in C$​, if the number of keys s.t. $E(k,m)=c$ is $\lambda$,
$$
Pr_{k \overset{R}{\leftarrow}K}[E(k,m)=c]=\frac{\lambda}{|K|}=const
$$
It’s trivial to say that for OTP, num of keys that map m to c is only 1.$\ \ \ \Box$

The bad news…

Are there other ciphers that has perfect secrecy and possibly have much, much shorter keys?

The bad news is that Shannon has proved it’s impossible.

Thm: If a cipher has perfect security, then $|K| \geq |M|$​.

Pseudo Random Generator (PRG)

How to make OTP into a practical scheme, namely using short key to guarantee “security”? The core of stream cipher is its key is not truly-random but pseudo-random.

Def: PRG is a func. $G:\{0,1\}^s \longrightarrow\{0,1\}^n,n>>s$ that map from seed space to a n-bit string and is “efficiently” computable by det. alg.

Comment:

  1. G is definitely deterministic.
  2. The output should look random.
  3. PRG must be unpredictable.

Stream Cipher

Let’s introduce our leader Stream Cipher:
$$
c=E(k,m):=m \oplus G(k)
$$

$$
D(k,c)=G(k)\oplus c\ \ \ (1.4)
$$

Can a stream cipher have perfect secrecy? No, since teh key must be as long as the message!

So right now, we need a different definition of security and it will depend on specific PRG.

PRG must be unpredictable

Suppose PRG is predictable,

$$ \exists i:G(k)|_{1,...,i}\overset{alg.}{\rightarrow}G(k)|_{i+1,...,n} $$

then the stream cipher is insecure. Why?

Just by some prior knowledge, the attacker actually knows the initial part of the message happens tobe some known value. For example, we know in the mail protocol, SMTP, the standard mail call used in the internet, every msg starts with a word from: colon. That would be a prefix that the adversary knows. So…
$$
c\oplus m_{1,…,i}=G(k)|_{1,…,i}
$$
the PRG is cracked!

Def: We say that $G:k\longrightarrow\{0,1\}^n$​ is predictable if

$\exists$ “eff” alg. A and $\exists 1\leq i\leq n-1$ s.t.

$$ Pr_{k \overset{R}{\leftarrow}K}[A(G(k)|_{1,...,i})=G(k)|_{i+1}]\geq \frac{1}{2}+\epsilon\ \ \ (1.5) $$

for some non-neg $\epsilon\geq1/2^{30}$

Quiz:

  • Is G: XOR(G(k)) = 1 predictable? Yes, given the first (n - 1) bits can predict the n’th bit.

Neg

In practice: $\epsilon$ is a scalr and

$\epsilon$ non-neg: $\epsilon\geq1/2^{30}$ likely to happen over 1GB data

$\epsilon$ neg: $\epsilon\leq1/2^{80}$​ won’t happen over life of key

In theory: $\epsilon$ is a function $\epsilon:Z^{\geq0}\longrightarrow R^{\geq0}$ and

$\epsilon$ non-neg: $\exists$d: $\epsilon(\lambda)\geq1/\lambda^b$ inf.often ($\epsilon\geq$1/poly, for many $\lambda$​)

$\epsilon$ negligible: $\forall d:\lambda\geq \lambda_d:\epsilon(\lambda)\leq1/\lambda^d$($\epsilon\leq$1/poly, for large $\lambda$)

$$
r[i]\longleftarrow a*r[i-1]+b\ \ \ mod\ p
$$

$$
r[0]\equiv seed
$$

and outputs few bits of $r[i]$​ then i++.

In fact, systems like Kerberos version four have used and been beaten by that.

PRG Security Defs

Let $G: k \longrightarrow \{0, 1\}^n$ be a PRG, our goal is to define what it means that $G(k)$ is “indistinguishable” from $r$.

Remember that,

$G(k)$ is the output when $k \overset{R}{\leftarrow}K$, and $r$ is the output when $r\overset{R}{\leftarrow}\{0, 1\}^n$.

Is this possible? Btw, the size of $\{0, 1\}^n$ is much much more larger than $G(k)$.

Statistical Tests

statistical test on $\{0, 1\}^n$: an alg $A$ s.t. $A(x$) outputs “0” or “1”, 0 means that the test thinks the input you give looks not random, well 1 means the input is random.

Examples:

  1. $A(x) = 1$ iff $|$#$0(x) -$ #$1(x)|\leq 10\cdot \sqrt{n}$
  2. $A(x) = 1$ iff $|$#$00(x) - \frac{n}{4}|$ $\leq 10 \cdot \sqrt{n}$

More examples:

  1. $A(x) = 1$ iff max-run-of-$0(x)$ $\leq 10\cdot \log_2(n)$

$\vdots$

Advantage

Let $G: k \longrightarrow \{0, 1\}^n$ be a PRG and $A$ a stat. test on $\{0, 1\}^n$, we define:

$$
Adv_{PRG}[A, G]=|Pr_{k \overset{R}{\leftarrow}K}[A(G(k))=1]-Pr_{r\overset{R}{\leftarrow}{0,1}^n}[A(r)=1]|\in[0,1]
$$
Comment:

Adv close to 1 —— A can dist. G from rand;

Adv close to 0 —— A cannot dist. G from rand.

A silly example: $A(x) = 0 \Rightarrow Adv_{PRG}[A, G] = |0-0|=0$

An interesting example:

Suppose $G: k \longrightarrow \{0, 1\}^n$ satisfies $msb(G(k)) = 1$ for 2/3 of keys in $K$, define stat.test $A(x)$ as:

if $msb(x) = 1$ output “1” else output “0”

Then

$Adv_{PRG}[A, G] = |Pr[A(G(k)=1)]-Pr[A(r)=1]| = |\frac{2}{3}-\frac{1}{2}|=\frac{1}{6}$

$msb$ means the most significant bit of the string, the first bit of the output.

this basically means A breaks G with advantage 1/6 .

Secure PRGs: crypto definition

Def: We say that $G: k \longrightarrow \{0, 1\}^n$ is a secure PRG if

for all “eff” stat. tests A: $Adv_{PRG}[A, G]$ is “neg”

Are there provable secure PRGs? Sorry, we cannot! P = NP?

if we can prove P = NP, it’s easy to show there are no secure PRGs;

if P is not equal to NP, you could prove to me a particular PRG is secure.

but we have heuristic candidates.

Easy fact: a secure PRG is unpredictable

We show: PRG predictable means PRG is insecure

Suppose A is an eff alg s.t.

$$ Pr_{k \overset{R}{\leftarrow}K}[A(G(k)|_{1,...,i})=G(k)|_{i+1}]=\frac{1}{2}+\epsilon $$

for non-negligible $\epsilon$ e.g. $\epsilon$ = 1/1000

Define stat. test B as:

B(x) outputs “0” or “1”, 1 means that $A(x_{1,…,i})=x_{i+1}$ .

如果一个PRG是可预测的，那么这个PRG是不安全的。也就意味着其逆否命题——一个安全的PRG一定是不可预测的——是正确的。

假设我们有一个有效的预测算法 A ，使得：

$$ Pr_{k \overset{R}{\leftarrow}K}[A(G(k)|_{1,...,i})=G(k)|_{i+1}]=\frac{1}{2}+\epsilon $$

对于一个真正随机的字符串，已知前 i 位成功预测下一位的概率是1/2 . 这是由于每一位要么是0要么是1. 如果对于算法 A ，能够以一个不可忽略的$\epsilon$，例如$\epsilon = 1/1000$ 成功预测下一位，那么这就是个危险的预测算法。

下面我们定义一个新的统计测试 B ，以字符串 x 作为输入，输出 0 或者 1. 如果 A 根据前缀正确预测下一位，即 $A(x_{1,…,i})=x_{i+1}$，那么$B(x)=1$.

对于一个真正随机的字符串$r\overset{R}{\leftarrow}\{0, 1\}^n$，有$Pr[B(r)=1]= \frac{1}{2}$ 成立。换句话说，算法 A 不提供第 i + 1 位的任何信息。

$$
Adv_{PRG}[B, G] > \epsilon
$$
如果发生器 G 是安全的，那么不存在有效的算法 A 和好的统计测试 B 以不可忽略的优势赢下游戏。

1982年姚期智在论文 Theory and application of trapdoor functions 中用熵的观点给出了一个十分优雅且重要的定理，即上述定理的逆命题也是正确的：如果发生器是不可预测的，那么这个发生器就是安全的。

Basic Key Exchange

——这一章将带领我们走进公钥密码的大门。

problem statement (22 mins)

Trusted 3rd Parties (11 mins)

通过前面对称密码的学习，我们了解到了流密码、分组密码和哈希函数等密码原语及协议的使用。如何安全地将密钥共享给通信的双方，成为实现这些的加密算法的关键前提。

假如现在有 n 个 users，例如 n = 4，这四个人要想安全地相互通信，每个人至少必须要记住 3 个密钥。用户 1 需要记忆 $k_{12}、k_{13}$ 和 $k_{14}$，其它 users 同理，这样会使得通信很麻烦。

如果我们引进一个可信任的第三方 (TTP)，也就是 trusted 3rd party。这个可信第三方和我们共享每个人各自的密钥 $k_1、k_2…k_n$，这样每个用户只需记忆自己单独一个密钥即可。

那么问题来了，如何使用这些密钥来实现双方的安全通信呢？这里我们先讨论最简单的情况，仅实现窃听安全 (eavesdropping security)，而不考虑攻击者的主动攻击。也就是说，我们需要实现一种协议，达到双方实现密钥共享，而攻击者不会获得密钥。

显然，在密钥交换的过程中仍然需要使用到对称密码，例如 AES。假设 Alice 试图和 Bob 进行密钥交换，Alice 的密钥是 $k_A$，Bob 的密钥是 $k_B$，记 “This is our shared key” 为消息 m。协议的内容是这样的：Alice 向可信第三方发送讯息m，然后TTP会生成一个$k_{AB}$，并将$E(k_A,m||k_{AB})$ 和一个 ticket = $E(k_B,m||k_{AB})$发送给Alice。Alice得到TTP的回复用$k_A$解密出$k_{AB}$，然后将ticket传递给Bob，Bob利用$k_B$解密出共享密钥。这里的E和D都是CPA选择明文安全的，也就意味着窃听者不能真正区分$k_{AB}$的加密和一个随机的字符串的加密。

事实上，使用这种TTP会有两个不可避免的缺点：1.必须在TTP在线时，双方才能交换密钥，这是一个交互式协议；2.一旦TTP被攻击者摧毁，由于TTP拥有所有的密钥，这种协议便不再安全。

Kerberos system

显然，这种协议对于主动的replay attack即重放攻击是完全insecure的。例如，攻击者记下了顾客Alice传递给商家Bob的内容，“Alice想从Bob那里订购一批书”，攻击者就可以反复使用这段密文，Bob就会一直认为Alice在进行购买。

我们能否设计出密钥交换协议对窃听和主动攻击都安全呢？Can we generate shared keys withot an online trusted 3rd party?

实际上这是可能的！这是公钥密码学的起点。Merkle(1974)，Diffle-Hellman(1976)，RSA(1977)以及ID-based enc.(BF 2001)，functional enc.(BSW 2011)。

Merkle Puzzles (11 mins)

简单起见，我们只允许窃听，而不允许任何对通信流量的篡改。实际上，我们可以仅仅使用对称密码，就可以不借助可信第三方实现密钥的交换。不幸的是我们的协议低效，不具备使用价值。

Puzzle: problems that is little hard but can be solved with some effort.

Alice准备$2^{32}$个puzzles，对于每个puzzle i，随机选择$p_i\in{0,1}^{32}$，$x_i、k_i\in{0,1}^{128}$.将$E(0^{96}||p_i,”Puzzle\ x_i”||k_i)$，作为谜题，将$2^{32}$个谜题发送给Bob。这个密钥的熵(entropy)是32位，$H(M)=-\sum prob_i/\log\ prob_i$。Bob随机选择其中一道谜题，花$2^{32}$的时间解出$x_i,k_i$，并将标记$x_i$发送给Alice，这样Alice就可以根据$x_i$找到相应的$k_i$​.

Alice和Bob的work都是$O(n)$，而eavesdropper则是$O(n^2)$​.这叫做quadratic gap，平方鸿沟。

but roughly speaking, quadratic gap is best possible if we treat cipher as a black box oracle [IR’89, BM’09]

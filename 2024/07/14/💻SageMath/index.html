Linux环境

这里我们使用wsl1在windows系统上使用linux。

  1. 启用wsl。
  2. 通过Microsoft Store下载安装Ubuntu，设置用户名和密码。
  3. 导出到D盘中：wsl --export Ubuntu D:\WSL\Ubuntu\ubuntu.tar。
  4. 注销并移除C盘中的发行版：wsl --unregister Ubuntu。
  5. 导入到D盘中：wsl --import Ubuntu D:\WSL\Ubuntu D:\WSL\Ubuntu.tar。

一些基础命令：sudo apt update/upgrade/install xxx/autoremove/clean/...。

下面我们准备编写python脚本的IDE。

  1. 在win中下载安装vscode，并安装名为wsl的插件。
  2. 在wsl中输入code .便可以打开vscode。
  3. 新建文件test.py，路径任意，例如/root/test.py。
  4. 写入文件，保存后新建终端，使用python3 test.py命令，得到输出。

下面我们说一说pip包管理器。

如果下载速度较慢，可以清除下载缓存，更换下载源。

  1. 创建虚拟环境：python3 -m venv myenv。

  2. 激活虚拟环境：source myenv/bin/activate。

  3. 安装包：pip install <package_name>。

  4. 退出：deactivate。

介绍一下ssh的方法clone一些github库：

  1. 创建ssh key：使用ssh-keygen -t rsa -b 4096 -C "your-email"命令，下面的选项全部直接敲回车即可。
  2. 随后使用 cat ~/.ssh/id_rsa.pub 命令查看生成的公钥，并完整的复制下来。
  3. 在github仓库界面点击自己的头像，选择settings。进入到设置页面后，点击左侧的SSH and GPG keys选项。点击New SSH key选项，并将复制下来的内容粘贴上去，添加该ssh key的描述。随后点击Add SSH key，并一路点击确认即可。
  4. 可以使用以下命令检查是否已有SSH密钥：ls -al ~/.ssh，如果你看到 id_rsa 和 id_rsa.pub 文件，说明已经存在SSH密钥。
  5. 使用以下命令测试与GitHub的SSH连接：ssh -T git@github.com

如果一切正常，你会看到类似以下的输出：

使用git clone git@github.com:username/repository.git克隆存储库。

按照文章一步一步来，编译时会禁root。

  1. 建立一个新用户：sudo adduser myuser
  2. 切换到新用户：su - myuser

当前myuser是没有sudo权限的。

或者也可以在Arch上安装SageMath。

但Arch的服务器比较慢，可以通过手动编辑镜像列表文件 /etc/pacman.d/mirrorlist，将速度较快的镜像服务器移到顶部。
